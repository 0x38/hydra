Config groups

As you continue experimenting, you realize you want to try several different optimizers. Let's add configuration for a second optimizer, adam.yaml and put both it and nesterov.yaml in an optimizer sub-directory:

$ tree 5_config_family/
examples/5_config_family/
├── README.md
├── conf
│   ├── config.yaml
│   └── optimizer
│       ├── adam.yaml
│       └── nesterov.yaml
└── experiment.py

optimizer/adam.yaml:

optimizer:
  type: adam
  lr: 0.1
  beta: 0.01

optimizer/nesterov.yaml:

optimizer:
  type: nesterov
  lr: 0.001

To load nesterov.yaml we can do as we did before:

defaults:
  - optimizer/nesterov

Running with this configuration, nesterov.yaml is loaded:

$ python experiment.py
optimizer:
  lr: 0.001
  type: nesterov

We can also do the this:

defaults:
  - optimizer: nesterov

Which is very similar, but allows us to do something quite nice:

$ python experiment.py optimizer=adam
optimizer:
  beta: 0.01
  lr: 0.1
  type: adam

Woa, we loaded optimizer/adam.yaml instead of optimizer/nesterov.yaml! We can still override individual values as before:

$ python experiment.py optimizer=adam optimizer.beta=0.1
optimizer:
  beta: 0.1
  lr: 0.1
  type: adam

Check the runnable example.

NOTE: This example shows a single config group, but you can easily add as many as you want.
